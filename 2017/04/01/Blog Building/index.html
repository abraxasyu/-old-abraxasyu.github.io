Blog Building

Published on Saturday, April 1st 2017 at 9:25
Last updated on Sunday, April 2nd 2017 at 21:26

Here, I will describe how to get a blog site like this up and running.

As with most web-related tech, there are a million options, ramifications of which are not entirely clear to amateur webdevs as myself, so I’ll try to explain why I made certain decisions.

Let me start by providing a very basic picture of webdev.

  • html - tells the browser what to display
  • css - enables you to better style your website
  • client-side scripting (javascript) - allows for interactive features
  • server-side scripting (e.g. PHP) - You may want a dynamic website where the client interacts with the server as opposed to the server simply producing websites for the client.
  • database - for certain websites, you may want to store some information in a database as opposed to storing them in volatile variables.
  • frameworks - streamlines some or all aspects of creating a website. Wordpress is the most famous of all blog frameworks.
  • hosting - in order to show your majestic creation to the world, it needs to be hosted somewhere. HostGator, GoDaddy, Bluhost etc. are the traditional web hosts, but there are alternatives. You could do cloud hosting on services such as AWS (amazon web services).
  • domain - provides the address for your website so users don’t have to type in the ip address.

So what does this website use?

This website was built using the hexo.io framework and is hosted on github pages through a custom domain.

Why github pages?

It’s free! Do note that there are certain limitations, the major one being that it can only host static sites. If I really wanted a dynamic site, I would’ve hosted on AWS which is also free as long as your resource usage remains below a certain threshold.

Why not wordpress?

Wordpress requires a MySQL or MariaDB database, whereas hexo.io is a static site generator. Using Wordpress means that I can’t host it on github pages.

Why not Jekyll or other SSG (static site generators)?

I’m familiar with Node.js and javascript, not Ruby. I really enjoy not having to switch between languages when dealing with client-side stuff and server-side stuff, so for simplicity’s sake, I really recommend learning Node.js.

So how do you do it?

Hexo has documentation, so consider this a primer. Once you have node.js, you can install hexo via npm. You can modify information in config.yml in the root directory. Files in the scaffold folder describes the default new page, post, etc. Source folder contains the actual blog content (in the markdown format). The theme folder contains the layout, theme-specific settings, scripts, etc.
